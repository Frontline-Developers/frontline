## 1. Project Overview

**Frontline** is a privacy-first, location-based citizen reporting and news platform. Users can:
- View a **Map Feed** of local events as interactive flutter_map markers with clustering
- Read a **Live Feed** combining citizen reports and GDELT wire news in a chronological split-timeline
- Submit **Anonymous Reports** via a multi-step form with camera/gallery media uploads
- Track their own submissions in a **My Reports** dashboard (local, sorted)
- **Confirm or Dispute** reports via a consensus/threshold voting system
- Leave **Comments** on reports (nested discussion threads)
- **Withdraw** their own reports at any time

**Core privacy guarantees (non-negotiable):**
- Coordinates are fuzzed Вұ3km server-side before any Firestore write
- No IP addresses logged anywhere вҖ” App Check enforces this
- Authentication is anonymous-only (Firebase Anonymous Auth)
- EXIF metadata stripped from all uploaded media via Cloud Functions + Sharp
- No PII stored: no names, emails, phones, or device identifiers

**Firebase project:** `frontline-549fb`
**Firestore region:** Singapore (`asia-southeast1`)
**Cloud Functions region:** Singapore (`asia-southeast1`)

## 3. Firestore Schema

### `reports/{reportId}`
```
userId:       string      вҶҗ anonymous Firebase UID (never linked to real identity)
location:     GeoPoint    вҶҗ fuzzed Вұ3km via fuzzReportLocation CF (NEVER raw coords)
geohash:      string      вҶҗ generated by geoflutterfire_plus for radius queries
category:     string      вҶҗ 'combat' | 'aid' | 'alert' | 'displaced' | 'infra' | 'other'
description:  string
mediaUrls:    string[]    вҶҗ Firebase Storage paths (EXIF stripped before upload)
status:       string      вҶҗ 'pending' | 'confirmed' | 'disputed' | 'withdrawn'
confirmCount: number      вҶҗ incremented by consensus CF
disputeCount: number      вҶҗ incremented by consensus CF
isDisputed:   boolean     вҶҗ set by spatial-temporal conflict CF (4.12)
exifStripped: boolean     вҶҗ true after client-side strip (CF re-verifies per task 5.1)
createdAt:    Timestamp
```

### `reports/{reportId}/interactions/{userId}`
```
type:         string      вҶҗ 'confirm' | 'dispute'
token:        string      вҶҗ unique vote token (prevents duplicate voting вҖ” task 5.8)
createdAt:    Timestamp
```

### `reports/{reportId}/comments/{commentId}`
```
userId:       string      вҶҗ anonymous UID
body:         string
createdAt:    Timestamp
```

### `wire_news/{articleId}`
```
title:        string
body:         string?
url:          string
source:       'wire'
publishedAt:  Timestamp
```
вҶҗ written only by `fetchGdeltNews` CF; client read-only

## 5. Key Feature Notes

### Map Feed (features/map)
- flutter_map + OpenStreetMap tiles вҖ” custom marker layers per category, optional clustering via `flutter_map_marker_cluster`. Picker (reporting flow) switched from Mapbox Flutter SDK after v2.24.3 was found to have no Flutter-web support (rendered blank canvas under DDC).
- GeoFire radius query (`geoflutterfire_plus`) on `reports` collection using bounding-box + geohash
- Filters by category and time range
- Tapping a cluster expands; tapping a marker opens Event Detail
- Tasks: 3.5, 3.6, 5.5

### Live Feed (features/feed)
- Split side-by-side chronological timeline: left column = citizen reports, right = wire news
- Pull-to-refresh, scrollable `ListView.builder`
- Citizen items from `reports/` (Firestore real-time listener), wire items from `wire_news/`
- Visually distinct card components per source type
- Tasks: 3.7, 3.8, 3.9, 5.4

### Anonymous Reporting (features/reporting)
- Multi-step form: location вҶ’ category вҶ’ description вҶ’ media upload вҶ’ review вҶ’ submit
- Camera/gallery via `image_picker` with permission handling
- Before submit: call `fuzzReportLocation` CF вҶ’ receive fuzzed coords вҶ’ write to Firestore
- Media upload to Storage (`reports/{userId}/{reportId}/{file}`) вҶ’ `stripExifMetadata` CF fires automatically
- Input validation at each step, guard against double-submit
- Tasks: 3.10, 3.11, 4.7, 5.1, 5.2, 5.3

### Event Detail (features/reporting вҖ” detail screen)
- Full-screen view of a single report
- Confirm / Dispute buttons вҶ’ call respective Cloud Functions
- Nested comment thread (Firestore real-time listener on `comments/`)
- Token-based voting to prevent duplicate confirms/disputes (task 5.8)
- "Disputed" banner if `isDisputed == true` (set by spatial-temporal conflict CF)
- Tasks: 3.12, 5.6, 5.7

### My Reports (features/my_reports)
- Queries `reports/` where `userId == currentUser.uid`
- Local sort by date / status
- "Withdraw" button вҶ’ calls `withdrawReport` CF вҶ’ deletes report + media
- Drift local cache for offline access (task 5.10) + retry queue (task 5.11)
- Tasks: 3.13, 5.9, 5.10, 5.11

### Consensus & Disputed Flags
- `confirmReport` / `disputeReport` CFs increment counts atomically
- Threshold logic in CF determines when `status` flips to 'confirmed' or 'disputed'
- `checkSpatialConflict` CF fires on new report creation вҖ” checks for spatially/temporally nearby conflicting reports and sets `isDisputed`
- Tasks: 4.11, 4.12

## 8. Additional Technical Decisions

### Offline Support (tasks 5.10, 5.11)
- `drift` package for local SQLite cache of user's own reports
- Offline action queue: if `submitReport` fails (no network), save to local queue and retry on reconnect
- Firestore SDK offline persistence handles read caching automatically

### EXIF Stripping (task 5.1)
- All media uploads go to Storage вҶ’ `onObjectFinalized` CF fires вҶ’ `sharp` library strips EXIF вҶ’ overwrites file
- Flutter client must NOT display media until CF confirms strip is complete (check a `exifStripped: true` field on the report doc)

### Consensus Thresholds (tasks 4.11, 5.6)
- Exact thresholds TBD by architect вҖ” placeholder: 3 confirms = 'confirmed', 3 disputes = 'disputed'
- Token-based: each user gets one vote per report; token stored in `interactions/{userId}`; CF checks existence before incrementing

### "Disputed" Flag Logic (task 4.12)
- Spatial: report within X km of an existing conflicting report
- Temporal: within Y hours of the conflicting report
- If both conditions met, `isDisputed` = true; shown as warning banner in Event Detail

### App Check (task 4.8)
- Android: Play Integrity API
- Web: reCAPTCHA v3
- Enforced on all callable Cloud Functions
- Purpose: prevent unauthorized API calls and ensure no IP logging via unregistered clients

### Map Tiles
- flutter_map fetches OpenStreetMap tiles via `https://tile.openstreetmap.org/{z}/{x}/{y}.png` вҖ” no API key required.
- `userAgentPackageName: 'app.frontline.mobile'` is set on the `TileLayer` per OSM tile-usage policy.
- For higher-volume usage, consider switching to a Mapbox raster tile URL with a token at build time
- Retrieved in Dart via `const String.fromEnvironment('MAPBOX_ACCESS_TOKEN')`
- Never hardcoded in source files
