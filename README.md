# Frontline

A privacy-first, location-based citizen reporting and news platform. Submit anonymous reports, view local events on an interactive map, and read a live feed combining citizen submissions with GDELT wire news вЂ” all without giving up your identity or exact location.

---

## Features

- **Map Feed** вЂ” Interactive Mapbox map with real-time citizen reports, custom SVG markers, and dynamic clustering
- **Live Feed** вЂ” Chronological split-timeline of citizen reports and GDELT wire news
- **Anonymous Reporting** вЂ” Multi-step form with camera/gallery uploads; coordinates fuzzed В±3km server-side before any storage
- **Confirm / Dispute** вЂ” Community consensus system with duplicate-vote protection
- **My Reports** вЂ” Personal submission history with offline support
- **Privacy by design** вЂ” Anonymous Firebase Auth, no PII stored, EXIF stripped from all media, no IP logging

## Getting Started

### 1. Clone

```bash
git clone https://github.com/Frontline-Developers/frontline.git
cd frontline
```

### 2. First-time setup

```bash
./setup.sh          # Linux / macOS
.\setup.ps1         # Windows (PowerShell)
```

The setup script:
- Checks all prerequisites
- Runs `flutter pub get` and `dart run build_runner build`
- Installs Cloud Functions npm dependencies
- Creates `apps/mobile/.env` from the example file
- Prompts for your **Mapbox access token** and saves it to `.env`
- Prompts for your **Firebase project ID** and runs `flutterfire configure`

### 3. Start developing

```bash
./dev.sh                  # Emulators + Flutter (auto-detect device)
./dev.sh --web            # Emulators + Flutter on Chrome
./dev.sh --prod           # Live Firebase + Flutter (Android)
./dev.sh --prod --web     # Live Firebase + Flutter (Chrome)
./dev.sh --emulator-only  # Start emulators only (for CF integration tests)
```

Windows:
```powershell
.\dev.ps1 [--web] [--prod] [--emulatorOnly]
```

Emulator UI в†’ http://127.0.0.1:4000

## Development Guide

### Architecture rules

- **Domain layer** imports nothing external вЂ” zero Flutter, zero Firebase
- **Firebase SDK** is only called inside `datasources/` files
- **Business logic** lives in domain UseCases, not in Notifiers or Screens
- New feature? Copy the pattern from `features/auth/` as the canonical reference

### State management

All features use the Riverpod `Notifier` pattern:

```dart
// Sentinel pattern for nullable copyWith fields
class FooState {
  final FooStatus status;
  final String? error;
  const FooState({this.status = FooStatus.idle, this.error});

  FooState copyWith({FooStatus? status, Object? error = _sentinel}) {
    return FooState(
      status: status ?? this.status,
      error: error == _sentinel ? this.error : error as String?,
    );
  }
}
const _sentinel = Object();
```

### Running tests

```bash
# Feature tests only
cd apps/mobile && flutter test test/features/<name>/ --no-pub

# Full suite
cd apps/mobile && flutter test --no-pub

# Cloud Functions (requires emulators running)

cd functions && npm test   # terminal 2
```

### Test-Driven Development

All features are developed test-first. Before writing any implementation code, use the `/tdd-feature` skill in Claude Code:

1. Write all tests for the feature
2. Run them вЂ” confirm every test fails
3. Implement until all tests pass
4. Never modify a test to make it pass вЂ” only fix implementation

See [`.claude/skills/tdd-feature/SKILL.md`](.claude/skills/tdd-feature/SKILL.md) for the full workflow.

### Code generation

After modifying any `@freezed` model or `@riverpod` provider:

```bash
cd apps/mobile && dart run build_runner build --delete-conflicting-outputs
```

### Privacy rules (non-negotiable)

1. **Location fuzzing** вЂ” all submitted coordinates must go through the `fuzzReportLocation` Cloud Function before any Firestore write. Never write raw `GeoPoint` from user input.
2. **No PII** вЂ” no names, emails, phone numbers, or IP addresses stored anywhere
3. **Anonymous auth only** вЂ” never prompt users for personal information
4. **EXIF stripping** вЂ” all uploaded media is processed by `stripExifMetadata` CF before being served
