# Frontline

A privacy-first, location-based citizen reporting and news platform. Submit anonymous reports, view local events on an interactive map, and read a live feed combining citizen submissions with GDELT wire news — all without giving up your identity or exact location.

---

## Features

- **Map Feed** — Interactive flutter_map/OpenStreetMap map with real-time citizen reports, radius filter, category/time filters, and "You are here" GPS toggle
- **Live Feed** — Combined timeline of citizen reports and GDELT wire news with inline vote counts and comment previews
- **Anonymous Reporting** — Multi-step form (describe → location → evidence); coordinates fuzzed ±3km server-side before any storage; up to 5 photos with per-photo remove
- **Confirm / Dispute** — Community consensus system; Cloud Functions atomically update vote counts and flip report status at threshold
- **Comments** — Threaded discussion on each report; pinned comments; sort by newest / top / oldest
- **Search** — Full-text AND keyword search across titles, body, location, category; recent searches; "What's Going On" top-5 trending countries
- **Compare** — Groups reports and wire news by category + date into SUPPORTS / CONTRADICTS / UNVERIFIED evidence timelines
- **Alerts** — Subscribe to a location + category combo; FCM push dispatched by Cloud Function on new matching reports
- **Bookmarks** — Save any feed item for later
- **PIN Gate** — Mandatory 6-digit PIN on every launch; biometric unlock opt-in (Android); "Forgot PIN" wipes all local data
- **My Reports** — Personal submission history with status filter
- **Privacy by design** — Anonymous Firebase Auth, no PII stored, EXIF stripped from all media, no IP logging

## Getting Started

### 1. Clone

```bash
git clone https://github.com/Frontline-Developers/frontline.git
cd frontline
```

### 2. First-time setup

```bash
./setup.sh          # Linux / macOS
.\setup.ps1         # Windows (PowerShell)
```

The setup script:
- Checks all prerequisites
- Runs `flutter pub get` and `dart run build_runner build`
- Installs Cloud Functions npm dependencies
- Creates `apps/mobile/.env` from the example file
- Prompts for your **Firebase project ID** and runs `flutterfire configure`

### 3. Start developing

```bash
./dev.sh                  # Emulators + Flutter (auto-detect device)
./dev.sh --web            # Emulators + Flutter on Chrome
./dev.sh --prod           # Live Firebase + Flutter (Android)
./dev.sh --prod --web     # Live Firebase + Flutter (Chrome)
./dev.sh --emulator-only  # Start emulators only (for CF integration tests)
```

Windows:
```powershell
.\dev.ps1 [--web] [--prod] [--emulatorOnly]
```

Emulator UI → http://127.0.0.1:4000

## Development Guide

### Architecture rules

- **Domain layer** imports nothing external — zero Flutter, zero Firebase
- **Firebase SDK** is only called inside `datasources/` files
- **Business logic** lives in domain UseCases, not in Notifiers or Screens
- New feature? Copy the pattern from `features/auth/` as the canonical reference

### State management

All features use the Riverpod `Notifier` pattern:

```dart
// Sentinel pattern for nullable copyWith fields
class FooState {
  final FooStatus status;
  final String? error;
  const FooState({this.status = FooStatus.idle, this.error});

  FooState copyWith({FooStatus? status, Object? error = _sentinel}) {
    return FooState(
      status: status ?? this.status,
      error: error == _sentinel ? this.error : error as String?,
    );
  }
}
const _sentinel = Object();
```

### Running tests

```bash
# Feature tests only
cd apps/mobile && flutter test test/features/<name>/ --no-pub

# Full suite
cd apps/mobile && flutter test --no-pub

# Cloud Functions (requires emulators running)
./dev.sh --emulator-only   # terminal 1
cd functions && npm test   # terminal 2
```

### Test-Driven Development

All features are developed test-first. Before writing any implementation code, use the `/tdd-feature` skill in Claude Code:

1. Write all tests for the feature
2. Run them — confirm every test fails
3. Implement until all tests pass
4. Never modify a test to make it pass — only fix implementation

See [`.claude/skills/tdd-feature/SKILL.md`](.claude/skills/tdd-feature/SKILL.md) for the full workflow.

### Code generation

After modifying any `@freezed` model or `@riverpod` provider:

```bash
cd apps/mobile && dart run build_runner build --delete-conflicting-outputs
```

### Privacy rules (non-negotiable)

1. **Location fuzzing** — all submitted coordinates must go through the `fuzzReportLocation` Cloud Function before any Firestore write. Never write raw `GeoPoint` from user input.
2. **No PII** — no names, emails, phone numbers, or IP addresses stored anywhere
3. **Anonymous auth only** — never prompt users for personal information
4. **EXIF stripping** — all uploaded media is processed by `stripExifMetadata` CF before being served

### Adding a new feature

Use the `/feature-scaffold` skill in Claude Code to generate the full Clean Architecture skeleton, then follow the TDD workflow:

```
# In Claude Code
/feature-scaffold
/tdd-feature
```

### Commit convention

[Conventional Commits v1.0.0](https://www.conventionalcommits.org/)

```
feat(map): add GeoFire radius query to map datasource
fix(reporting): call fuzzReportLocation CF before Firestore write
test(feed): add widget tests for empty and error states
chore(deps): upgrade flutter_map to 8.1.0
```

Types: `feat` · `fix` · `docs` · `test` · `refactor` · `chore` · `ci` · `perf` · `revert`

### Branch workflow

```
main              ← protected, PRs only, humans merge
  └── feat/<name>
  └── fix/<name>
  └── chore/<name>
```

Never push directly to `main`. Never merge your own PR.

## Environment Variables

Copy `apps/mobile/.env.example` to `apps/mobile/.env` (gitignored):

# true → local emulators | false → production Firebase

This value is passed to Flutter via `--dart-define` and never bundled in committed source. OpenStreetMap tiles require no access token.

## Firebase Setup

New developer onboarding:

1. Run `./setup.sh` — it will prompt for your Firebase project ID
2. Or manually: `cd apps/mobile && flutterfire configure --project=frontline-549fb --platforms=android,web`

This generates `apps/mobile/lib/firebase_options.dart` and `apps/mobile/android/app/google-services.json` (both gitignored — each developer generates their own).

## CI / CD

Two GitHub Actions workflows run on every PR:

| Workflow | Triggers | Jobs |
|---|---|---|
| `ci.yml` | push + PR to `main` | `flutter-quality` (analyze · test) · `functions-quality` (lint · build) |
| `build.yml` | PR to `main` (non-markdown) | `build-android` (debug APK) · `build-web` |

| Secret | Description |

| `FIREBASE_OPTIONS` | `base64 -w 0 apps/mobile/lib/firebase_options.dart` |
